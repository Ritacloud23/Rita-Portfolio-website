Medplus DevOps Deployment

End-to-end automation of the Medplus web application — containerized, pushed to ECR, and deployed to AWS EC2 with CI/CD and Portainer.

  1. Home
  2. Medplus Project

Project Information

  • Category: Cloud & DevOps Automation
  • Project Date: October 2025
  • Stack: AWS, Docker, GitHub Actions, ECR, EC2, Portainer
  • GitHub Repo: github.com/Ritacloud23/medplus

Project Overview

The Medplus DevOps Deployment demonstrates a full CI/CD pipeline where the application is containerized, tested, and deployed automatically to AWS. The project integrates multiple AWS services with Docker, GitHub Actions, and Portainer for end-to-end automation.

Deployment Steps

  1. Containerization: Built a lightweight Docker image using a python:3.11-slim base, optimized with multi-stage builds.
  2. ECR Integration: Created a private AWS ECR repository and used aws ecr get-login-password for authentication.
  3. CI/CD Pipeline: Automated build, tag, and push workflows using GitHub Actions.
  4. EC2 Deployment: Pulled and deployed the ECR image to an EC2 instance via Portainer for visual container management.
  5. Reverse Proxy Setup: Configured NGINX as a reverse proxy and applied SSL using Certbot.

Outcome

The project achieved zero-downtime deployment with secure, automated workflows from code commit to production deployment. It highlights key DevOps principles of automation, scalability, and monitoring in real-world AWS environments.
